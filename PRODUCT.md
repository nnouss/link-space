# Product

## Register

product

## Users

Link Space is used by people who need to analyze how a search or browsing session unfolded. They are reviewing paths, branches, and relationships between visited pages rather than simply saving links for later.

## Product Purpose

Link Space captures browsing/search sessions and presents them as an explorable link graph. The dashboard should help users understand the structure of an exploration: where it started, where it branched, which pages mattered, and how pages relate to one another.

Success means a user can open the dashboard and quickly read the shape of a session before drilling into node-level details.

## Brand Personality

Precise, calm, premium.

The interface should feel analytical and composed. It should support close inspection without becoming visually noisy, and it should make the graph feel like an instrument rather than decoration.

## Anti-references

Avoid flat, generic productivity UI that makes the graph feel secondary. Avoid exaggerated 3D spectacle, neon hacker styling, or game-like visuals that distract from analysis.

## Design Principles

1. Put the graph first. The dashboard should preserve as much meaningful space as possible for the session graph.
2. Make structure readable. Panels, labels, and controls should clarify branches, counts, status, and selected-node context.
3. Keep the surface composed. Use restraint, hierarchy, and detail rather than loud effects.
4. Support progressive inspection. Start with the session shape, then reveal detail when a session or node is selected.
5. Use copy as instrumentation. Labels should be clear, compact, and analysis-oriented.

## Accessibility & Inclusion

Target WCAG AA contrast for interface text and controls. Do not communicate state through color alone. Respect reduced-motion preferences for interface transitions where practical.
